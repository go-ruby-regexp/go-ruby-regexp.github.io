go-onigmo is a pure-Go (no cgo) reimplementation of Onigmo, the regular-expression engine used by Ruby. Go's standard regexp is RE2 — linear-time but without backreferences or lookaround, and with different match semantics — so a byte-compatible Ruby regexp needs a backtracking engine. go-onigmo is that engine: a faithful backtracking VM, hardened against catastrophic backtracking with memoization and a deterministic time/step budget. It is standalone and reusable, and is the regexp backend for go-embedded-ruby.

A faithful backtracking VM in pure Go, cgo disabled, so it cross-compiles and embeds anywhere. It implements the Onigmo features RE2 omits — backreferences, lookaround, possessive quantifiers, atomic groups, named groups, subexpression calls — with Ruby's leftmost-first semantics, and is hardened against ReDoS with memoization and a deterministic budget (as Ruby ≥3.2). Validated differentially against Onigmo/MRI. It is a standalone, reusable module and the regexp backend for the sibling org github.com/go-embedded-ruby.
